## 1. Introduction

### 1) Summary

오프라인 은행 창구는 여전히 많은 고객이 이용하는 핵심 대면 채널이지만, 행원들은 직급에 따라 권한과 지식이 달라 매번 고객의 요구를 빠르게 처리하기 어렵다. 또한 기존 번호표 시스템은 고객이 직접 업무 종류를 선택해야 하므로 오류가 잦고, 적절하지 않은 창구로 배정되는 경우가 빈번하다. 한편 온라인 채널에서는 너무 많은 금융 상품 정보가 나열되어 있어 고객 개개인에게 맞는 상품을 찾기 어렵다.

본 프로젝트 'AI BankPass'는 이러한 온·오프라인의 불편함을 하나의 통합 웹 플랫폼으로 해결하고자 한다. 별도의 앱 설치 없이 웹 경로 분리를 통해 고객용 서비스와 지점 키오스크 서비스를 동시에 제공하며, AI를 통해 개인화된 상품 추천(Cosine Similarity)과 지능형 창구 배정(Random Forest) 기능을 제공한다. 이를 통해 고객은 본인에게 맞는 금융 상품을 쉽게 찾을 수 있고, 지점 방문 시 적절한 창구로 자동 안내받을 수 있다. 행원은 자신에게 배정된 대기 고객 정보를 사전에 파악할 수 있어 업무 효율성이 향상된다.

### 2) Describe the features of project

'AI BankPass'는 통합 웹 플랫폼을 바탕으로 고객에게 AI 기반 맞춤형 금융 상품 추천, 상품 상세 조회, AI 지능형 창구 접수 서비스를 제공하고, 행원에게는 대기열 관리 및 고객 호출 기능을 제공하는 시스템이다.

## 3. Domain Analysis

### 1) Main_BankPass

시스템의 모든 과정이 이 클래스를 통해 시행된다. 모든 클래스의 attribute와 operation은 이 클래스를 통해 실행 및 변경이 이루어진다. 고객용 웹 경로와 키오스크용 웹 경로, 행원용 웹 경로를 분리하여 라우팅한다.

### 2) Registration

시스템을 이용하기 위한 권한을 부여하는 클래스이다. Customer(권한 코드 0)와 Banker(권한 코드 1)로 나누어 가입하게 한다. 이 클래스를 통해 회원가입한 사용자만 로그인 기능을 사용할 수 있다.

### 3) Login

시스템을 사용하기 위해 실행해야 하는 클래스이다. 시스템에 접근하려는 사용자의 ID와 Password를 검증하고, 권한 코드에 따라 적절한 메인 화면으로 라우팅한다.

### 4) Recommendation

Customer가 로그인했을 때 자동으로 실행되는 클래스이다. 고객의 금융 데이터를 Python AI 서버로 전송하고, Cosine Similarity 알고리즘을 통해 반환받은 Top 3 상품을 화면에 렌더링한다. AIEngine 클래스와 연동된다.

### 5) ProductInquiry

Customer가 추천 상품 또는 전체 상품 리스트에서 특정 상품을 선택했을 때 상세 정보를 보여주는 클래스이다. Product 클래스에서 데이터를 가져와 화면에 출력한다.

### 6) Product

금융 상품의 정보(상품명, 이자율, 기간, 우대 조건, 가입 조건 등)를 저장하는 클래스이다. ProductInquiry와 Recommendation 클래스에 정보를 제공한다.

### 7) SmartTicket

Customer가 키오스크에서 접수 버튼을 눌렀을 때 실행되는 클래스이다. AIEngine의 Random Forest 모델을 호출하여 방문 목적을 분류하고, 적절한 창구의 Queue에 번호를 등록한 뒤 종이 번호표를 출력한다.

### 8) AIEngine

시스템의 AI 기능을 담당하는 클래스이다. FastAPI/Flask 기반으로 서빙되며, Cosine Similarity 모델(상품 추천)과 Random Forest 모델(방문 목적 분류)을 제공한다. Recommendation과 SmartTicket 클래스로부터 호출된다.

### 9) Queue

번호표를 발급받은 고객들이 순서대로 상담을 기다리는 데이터의 나열 상태를 관리하는 클래스이다. 창구별로 별도의 대기열을 가지며, 각 대기표의 상태(대기 / 호출됨 / 완료)를 추적한다.

### 10) QueueManagement

Banker가 자신의 창구 대기열을 관리하는 클래스이다. Queue 클래스에서 대기 고객 목록을 가져오고, 호출 및 완료 처리를 통해 Queue 상태를 갱신한다.

### 11) Notification

호출 발생 시 고객에게 알림을 전달하는 클래스이다. QueueManagement 클래스로부터 호출되어 실시간 알림을 처리한다.

### 12) Database

Customer, Banker, Product, Queue, Transaction(거래 이력) 등의 데이터를 저장하고 관리하는 클래스이다. 모든 다른 클래스가 이 Database를 통해 데이터에 접근한다.

```
1. 회원가입  /  2. 로그인

3. 추천 상품 보기  /  4. 전체 상품 조회  /  5. 상품 상세 조회
6. 키오스크 번호표 발급(매장 내)  /  7. 로그아웃
입력 : 2
```

- System이 시작되면 메인 화면이 출력된다. Customer와 Banker는 각자의 권한에 따라 접근할 수 있는 기능이 다르므로 로그인 후 사용할 수 있는 메뉴가 달라진다.

```
<< 회원 가입 >>

ID : Customer1
Password : happyday0506
Name : 홍길동

Email : customer1@naver.com
권한 코드 : 0  (0: 고객 / 1: 행원)
```

- 사용자가 회원가입을 할 때 실행되는 Interface이다. ID, Password, Name, Email, PhoneNumber, 권한 코드와 같은 사용자의 주요 정보를 입력하면 회원가입이 가능하다. 이때 ID는 중복될 수 없다.

```
<< Login >>

ID : Customer1
```

```
<< AI 추천 상품 >>

Customer1님을 위한 맞춤 추천 상품 (Cosine Similarity 기반)

[1순위] KB Star 정기예금
        - 이자율 : 연 3.8%   기간 : 12개월

[2순위] NH 더 행복한 적금
        - 이자율 : 연 4.2%   기간 : 24개월

[3순위] 청년 도약 정기예금
        - 이자율 : 연 4.5%   기간 : 36개월
=========================================================
```

- 고객이 로그인하면 AI 서버가 고객의 금융 데이터를 분석하여 가장 적합한 상품 Top 3를 추천한다. 클릭 시 상품 상세 페이지로 이동한다.

### 5) Inquire Product Details (상품 상세 조회)

```
<< 상품 상세 정보 >>
=========================================================
상품명     : KB Star 정기예금
이자율     : 연 3.8% (우대 시 최대 4.3%)
가입 기간  : 12개월
가입 조건  : 만 19세 이상
우대 조건  : 자동이체 등록 시 +0.3%, 모바일 가입 시 +0.2%
=========================================================
```

- Consumer는 추천 상품 또는 전체 상품 목록에서 특정 항목을 클릭하면 해당 상품의 상세 정보를 확인할 수 있다. 정보는 DB의 Product 테이블에서 가져온다.

### 6) Reserve Smart Ticket (AI 지능형 창구 접수)

```
<< 키오스크 번호표 발급 >>
=========================================================
[1] 접수 버튼을 누른다
    ↓
[AI 분석 중 ...]  ← Random Forest 모델이 방문 목적 예측

분석 결과 : 대출 상담
배정 창구 : 3번 창구 (대출 전담)

------ 종이 번호표 출력 ------
       AI BankPass
   ─────────────────
     대기번호 : 047
     배정창구 : 3번 (대출)
     앞 대기 인원 : 5명
     예상 대기시간 : 약 25분
   ─────────────────
=========================================================
```

- 고객이 키오스크에서 접수 버튼을 누르면 AI(Random Forest)가 방문 목적을 예측하고, 적합한 창구의 대기열에 번호를 발급한다. 키오스크에서 종이 번호표가 즉시 인쇄되어 출력된다.

### 7) Manage Queue (대기열 관리 - 행원용)

```
<< 3번 창구 대기열 관리 (담당: Banker1) >>
=========================================================
순번  대기번호  고객명     예측 방문목적   상태
─────────────────────────────────────────────────────────
1     045      김OO       대출 상담        호출됨
2     047      Customer1  대출 상담        대기
3     048      이OO       대출 상담        대기
4     051      박OO       대출 연장        대기
─────────────────────────────────────────────────────────
[호출하기]   [완료처리]   [건너뛰기]
=========================================================
```

- Banker는 자신에게 배정된 창구의 대기 고객 목록을 실시간으로 확인할 수 있다. AI가 분석한 방문 목적을 미리 알 수 있어 상담을 효율적으로 준비할 수 있다. '호출' 버튼을 누르면 고객에게 알림이 전달되고, 상담이 끝나면 '완료' 버튼으로 상태를 업데이트한다.

## 5. Glossary

| Terms | Description |
|---|---|
| 고객 (Customer) | AI BankPass를 통해 금융 상품을 조회·추천받고, 영업점 방문 시 번호표를 발급받는 사용자 |
| 행원 (Banker) | 영업점에서 고객 대기열을 관리하고 상담을 진행하는 사용자 |
| AI BankPass | 본 프로젝트에서 개발하는 '통합 지능형 창구 라우팅 및 추천 시스템'의 공식 명칭 |
| 키오스크 (Kiosk) | 영업점 내 설치되어 고객이 직접 번호표를 발급받는 웹 인터페이스 단말기 |
| 대기열 (Queue) | 번호표를 발급받은 고객들이 순서대로 상담을 기다리는 데이터의 나열 상태 |
| 라우팅 (Routing) | AI 분석 결과에 따라 고객을 가장 적합한 창구의 대기열로 배정하는 과정 |
| Cosine Similarity (코사인 유사도) | 두 데이터 벡터 간의 코사인 각도를 이용하여 유사도를 측정하는 알고리즘. 맞춤형 금융 상품 추천에 사용 |
| Random Forest (랜덤포레스트) | 다수의 의사결정 나무를 앙상블하는 다중 분류 AI 모델. 본 시스템에서는 방문 목적 예측 및 창구 라우팅에 사용 |
| Mock Data (가상 데이터) | 실제 금융 데이터 접근이 불가능하므로 AI 모델 학습과 DB 구성을 위해 임의로 생성한 가상의 훈련용 고객 거래 데이터 |
| 권한 코드 | 사용자의 역할을 구분하는 값. 0은 고객, 1은 행원을 의미 |
