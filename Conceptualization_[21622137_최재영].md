## 1. Business purpose

### Project background

오프라인 은행 창구는 여전히 많은 고객이 이용하는 핵심 대면 채널로써 늘 분주한데, 행원들은 직급에 따라 권한이나 지식이 달라 다른 행원의 도움을 요청하는 등 매번 고객의 요구를 빠르게 처리하는데 있어 어려움을 겪고 있다. 반면 온라인 채널에서는 너무 많은 금융 상품 정보가 나열되어 있어 고객 개개인에게 맞는 상품을 찾기 어렵다.

본 프로젝트 'AI BankPass'는 이러한 온·오프라인의 불편함을 하나의 통합 웹 플랫폼으로 해결하고자 한다. 별도의 앱 설치 없이 웹 경로 분리를 통해 고객용 서비스와 지점 키오스크 서비스를 동시에 제공하며, AI를 통해 개인화된 상품 추천과 지능형 창구 배정 기능을 제공한다.

### Goal

첫째, 맞춤형 상품 추천: 고객의 금융 거래 이력 기반의 데이터(협업 필터링)를 활용하여 웹 접속 시 개인화된 금융 상품을 우선적으로 추천한다.

둘째, AI 지능형 라우팅: 영업점 키오스크(웹 인터페이스)에서 방문 접수 시 AI가 방문 목적을 예측하여 적절한 전담 창구로 번호표를 자동 발급한다.

셋째, 통합 관리 환경 구축: 하나의 백엔드 서버를 공유하는 통합 웹 시스템을 구축하여 데이터 정합성을 확보하고 개발 및 운영 효율성을 높인다.

### Target Market

금융 업무를 보는 '은행 고객'과 영업점 현장에서 고객 대기열을 관리하는 '은행 행원'이 주 대상자이다.

## 2. System context diagram

이 프로젝트를 위해 작성한 system context diagram의 구조는 아래와 같다.

```
                5. 로그인 요청                    1. 로그인/가입 요청
                7. 고객 표를 및 상태 변경 요청      3. 번호표 발급 요청
                        →                              →
<<actor>> Banker                  system                      <<actor>> Customer
  (행원/관리자)         ←        (서버 및 AI 엔진)      →           (고객)
                        
                8. 할당 창구 대기열 흐름 반환        2. 고객 결보, AI 추천 상품 반환
                9. 고객 상세 결보 및 예실 발른 흐름 반환  4. AI 배정 창구 및 대기 순번 반환
```

시스템의 중심에는 데이터를 처리하고 AI 모델(추천, 분류)을 서빙하는 System이 존재하며, 이를 통해 Customer(고객)와 Banker(행원) 간의 상호작용이 실시간으로 이루어진다.

## 3. Use case list

이 프로젝트에서 발견할 수 있는 use cases는 다음과 같다.

### 1) Register Member (회원가입)

| | |
|---|---|
| **Actor** | Customer, Banker |
| **Description** | 시스템을 이용하기 위해 등록되지 않은 소비자와 행원을 회원으로 등록한다. 고객은 0, 행원은 1의 값을 주어 권한을 구분한다. |

### 2) Login (로그인)

| | |
|---|---|
| **Actor** | Customer, Banker |
| **Description** | 각자 자신의 아이디와 비밀번호로 로그인한다. 시스템은 권한을 확인하고 성공 여부를 반환하여 직책에 맞는 기능을 제공한다. |

### 3) Recommend Financial Products (AI 금융 상품 추천)

| | |
|---|---|
| **Actor** | Customer |
| **Description** | 고객이 로그인하면 시스템(AI)이 고객의 금융 데이터를 분석하여(Cosine Similarity), 메인 화면에 가장 적합한 금융 상품들을 추천 목록으로 띄워준다. |

### 4) Inquire Product Details (상품 상세 조회)

| | |
|---|---|
| **Actor** | Customer |
| **Description** | 추천된 상품이나 전체 상품 리스트에서 특정 상품을 선택하여 금리, 기간, 가입 조건 등의 상세 정보를 조회한다. |

### 5) Reserve Smart Ticket (AI 지능형 창구 접수)

| | |
|---|---|
| **Actor** | Customer |
| **Description** | 고객이 키오스크에서 접수를 요청하면, AI(Random Forest)가 고객 데이터를 기반으로 방문 목적을 예측하여 적합한 창구(예: 대출창구, 예금창구)에 대기 번호를 자동 배정하고, 발급된 번호와 현재 대기 인원수가 인쇄된 종이 번호표를 기기에서 출력한다. |

### 6) Manage Queue (대기열 관리 및 고객 호출)

| | |
|---|---|
| **Actor** | Banker |
| **Description** | 행원은 자신의 창구에 배정된 대기 고객 목록을 확인한다. 고객을 '호출'하거나 상담을 마친 후 '완료' 상태로 변경하여 시스템의 대기열을 갱신한다. |

## 4. Concept of operation

이 장은 use cases를 어떻게 작동할지에 대한 구체적인 작동 개념을 기술한 장이다.

### 1) Register Member & Login

| | |
|---|---|
| **Purpose** | 시스템 내에서 고객의 금융 이력과 행원의 창구 권한을 명확히 식별하기 위해 고유한 사용자 ID가 필요하다. |
| **Approach** | 회원가입을 통해 DB에 기본 정보(ID, 암호화된 비밀번호, 고객/행원 권한 코드)를 저장한다. 로그인 시 ID와 Password를 비교 검증하여 권한에 따라 다른 메인 화면(고객용/행원용)으로 라우팅한다. |
| **Dynamics** | 앱을 처음 실행하여 서비스를 이용하고자 할 경우. |
| **Goals** | 안전한 인증 및 권한(Role)에 따른 기능 분리 제공. |

### 2) Recommend Financial Products

| | |
|---|---|
| **Purpose** | 방대한 금융 상품 중 고객이 필요로 할 만한 상품을 찾아주는 수고를 덜어주어 플랫폼 만족도를 높인다. |
| **Approach** | 고객이 로그인함과 동시에 서버는 해당 고객의 나이, 직업, 기존 예적금 금액 등의 데이터를 Python AI 추천 서버로 넘긴다. Cosine Similarity 알고리즘을 통해 가장 유사한 성향의 고객들이 가입한 상품 Top 3를 반환받아 화면에 렌더링한다. |
| **Dynamics** | 고객이 로그인 직후 메인 화면에 진입하는 경우. |
| **Goals** | 맞춤형 큐레이션을 통해 고객의 상품 탐색 시간을 단축한다. |

### 3) Inquire Product Details

| | |
|---|---|
| **Purpose** | 추천받은 상품에 대해 소비자가 정확한 판단을 내릴 수 있도록 객관적인 정보를 제공해야 한다. |
| **Approach** | 상품 리스트의 아이템을 터치/클릭하면 데이터베이스의 'Product' 테이블에서 해당 상품의 상세 정보(이자율, 기간, 우대 조건 등)를 가져와 상세 페이지를 보여준다. |
| **Dynamics** | 추천 상품이나 전체 상품 목록 중 관심 있는 항목을 클릭한 경우. |
| **Goals** | DB 조회를 통한 정확한 금융 정보 제공. |

### 4) Reserve Smart Ticket (AI 지능형 창구 접수)

| | |
|---|---|
| **Purpose** | 일반적인 방식인 번호표의 선택 오류를 줄이고, 업무 성격에 맞는 전문 창구로 고객을 자동 안내하기 위함이다. |
| **Approach** | 고객이 키오스크에서 접수 버튼을 누르면, Random Forest 모델이 가동되어 고객의 과거 금융 거래 데이터를 분석해 방문 목적(대출, 예금 등)을 분류한다. 해당 카테고리를 담당하는 창구의 대기열 테이블에 새로운 번호를 발급하며, 이때 대기 번호와 내 앞의 대기 인원수 정보가 담긴 종이 번호표를 키오스크에서 즉시 인쇄하여 출력해 준다. |
| **Dynamics** | 오프라인 지점을 방문하여 번호표를 뽑고자 하는 경우. |
| **Goals** | AI를 활용한 트래픽 분산 및 창구 라우팅의 자동화. |

### 5) Manage Queue

| | |
|---|---|
| **Purpose** | 행원이 대기 중인 고객의 정보를 파악하고 오프라인 상담을 순차적으로 진행하기 위함이다. |
| **Approach** | 행원으로 로그인하면 자신에게 할당된 창구 번호의 Queue 데이터를 리스트로 불러온다. 행원이 '호출' 버튼을 누르면 해당 대기표의 상태가 '호출됨'으로 바뀌며 고객에게 알림이 가고, 처리가 끝나면 '완료'로 상태를 DB에 업데이트한다. |
| **Dynamics** | 앞선 고객의 업무 처리가 끝나 다음 고객을 응대해야 하는 경우. |
| **Goals** | 혼선 없는 대기열 관리 및 AI가 분석한 고객 방문 목적의 사전 인지. |

## 5. Problem statement

이 장에서는 소프트웨어를 설계하고 구현하는 과정에서 고려해야 할 기술적 관점과 다양한 문제점(Rootcause)을 기술한다.

### AI 모델 연동 및 응답 속도

이 시스템의 핵심은 AI 기능이다. RandomForest 및 Cosine Similarity 모델은 Python 환경(Scikit-learn)에서 학습되었다. 이를 프론트엔드 시스템과 통신시키기 위해서는 모델을 피클 형태로 저장하고, FastAPI나 Flask를 이용해 API 형태로 서빙해야 한다. 이 과정에서 병목 현상이 발생하여 AI 응답이 지연되면 고객은 번호표 발급 시 무한 로딩을 경험할 수 있다. (해결책: 가벼운 모델 최적화 및 비동기 처리 적용)

### 가상 데이터의 한계

실제 은행의 고객 데이터는 개인정보보호법에 의해 접근이 불가능하다. 따라서 AI 모델을 학습시키기 위해 현실성 있는 대량의 '가상 마스터 데이터(MockData)'를 파이썬 스크립트로 구축하는 데 많은 설계 노력이 필요하다.

## 6. Glossary

| Terms | Description |
|---|---|
| Cosine Similarity (코사인 유사도) | 두 데이터 벡터 간의 코사인 각도를 이용하여 서로 얼마나 유사한지 측정하는 알고리즘으로, 맞춤형 금융 상품 추천에 사용된다. |
| Random Forest (랜덤포레스트) | 다수의 의사결정 나무(Decision Tree)를 앙상블하여 예측의 정확도를 높이는 다중 분류 AI 모델로, 본 시스템에서는 방문 목적 예측 및 창구 라우팅에 쓰인다. |
| AI 뱅크패스 (AI BankPass) | 본 프로젝트에서 개발하는 '통합 지능형 창구 라우팅 및 추천 시스템'의 공식 명칭. |
| 대기열 (Queue) | 번호표를 발급받은 고객들이 순서대로 상담을 기다리는 데이터의 나열 상태를 의미한다. |
| 라우팅 (Routing) | AI 분석 결과에 따라 고객을 여러 창구(예금, 대출, 외환 등) 중 가장 적합한 전담 창구의 대기열로 배정하는 과정. |
| Mock Data (가상 데이터) | 실제 금융 데이터 접근이 불가능하므로, AI 모델 학습과 DB 구성을 위해 임의로 생성한 가상의 훈련용 고객 거래 데이터. |
